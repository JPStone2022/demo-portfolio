Accessibility Statement

Julian Stone is committed to ensuring digital accessibility for people with disabilities. We are continually improving the user experience for everyone and applying the relevant accessibility standards.

Measures to Support Accessibility

This portfolio website takes the following measures to ensure accessibility:

  • Include accessibility as part of our design and development practices.
  • Use semantic HTML for clear structure and meaning.
  • Provide alternative text for images where appropriate.
  • Ensure sufficient color contrast for text elements.
  • Strive for keyboard navigability.
  • Employ responsive design for usability across various devices.

Conformance Status

The Web Content Accessibility Guidelines (WCAG) defines requirements for designers and developers to improve accessibility for people with disabilities. This site aims to adhere to WCAG 2.1 Level AA standards where possible.

While we strive to adhere to the accepted guidelines and standards for accessibility, it is not always possible to do so in all areas of the website, particularly with dynamically loaded content or third-party integrations.

Feedback

We welcome your feedback on the accessibility of this portfolio. Please let us know if you encounter accessibility barriers:

  • Via the Contact page.

We try to respond to feedback within a reasonable timeframe.

Technical Specifications

Accessibility of this portfolio relies on the following technologies to work with the particular combination of web browser and any assistive technologies or plugins installed on your computer:

  • HTML
  • CSS (Tailwind CSS)
  • JavaScript

These technologies are relied upon for conformance with the accessibility standards used.
